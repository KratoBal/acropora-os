# Security Policy

## Vulnerability reporting

Kérjük, feltételezett biztonsági sérülékenységet ne nyilvános GitHub issue-ban jelents.

Ha a repository GitHub-oldalán elérhető a **Security → Report a vulnerability** lehetőség, használd a GitHub privát vulnerability reporting folyamatát. Ha ez a lehetőség nem érhető el, ne ossz meg exploitot, titkot, személyes adatot vagy részletes támadási lépéseket nyilvános csatornán; várd meg, amíg a repository fenntartója biztonságos kapcsolattartási módot biztosít.

## Scope and response

A bejelentésben csak a reprodukcióhoz szükséges minimális információ szerepeljen. A javítás és a közzététel időzítését a fenntartóval összehangoltan kell kezelni. Biztonsági javítás előtt valós ügyféladat vagy production secret nem használható tesztelésre.

## Supported versions

A projekt jelenleg aktív fejlesztés alatt áll, formális támogatási mátrix még nincs. A legfrissebb alapértelmezett branch állapota az elsődleges vizsgálati cél.
